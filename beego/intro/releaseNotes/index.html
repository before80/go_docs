beego 2.0.0

Refactor 重构

  1. Support the new project structure. We split the whole framework into four parts: 1.1 server: including web module. 1.2 client: including ORM, cache, httplib modules. 1.3 task: supporting timed tasks and other cyclic tasks. 1.4 core: including validation, config, logs and admin modules. 支持新的项目结构。我们将整个框架分为四个部分：1.1 服务器：包括 web 模块。1.2 客户端：包括 ORM、缓存、httplib 模块。1.3 任务：支持定时任务和其他循环任务。1.4 核心：包括验证、配置、日志和管理模块。
  2. Add adapter module which is used to reduce the effort of upgrading Beego from v1.x to v2.x 添加 adapter 模块，用于减少将 Beego 从 v1.x 升级到 v2.x 的工作量
  3. Add context.Context for cache, httplib, session, task, ORM modules’ API. 为 context.Context 、 cache 、 httplib 、 session 、 task 、 ORM 模块的 API 添加
  4. Add error as a return value for cache, httplib, session, task. Now users are able to know more info about what happen inside Beego. 将 error 作为 cache 、 httplib 、 session 、 task 的返回值添加。现在用户能够了解有关 Beego 内部发生的事情的更多信息。
  5. Decouple modules from each other. All modules only depend on core package. 将模块彼此解耦。所有模块仅依赖于 core 包。
  6. Support tracing, metrics in ORM, web, httplib modules. 支持 ORM、web、httplib 模块中的跟踪、指标。
  7. Introduce filter-chain patter to support AOP. 引入 filter-chain 模式以支持 AOP。
